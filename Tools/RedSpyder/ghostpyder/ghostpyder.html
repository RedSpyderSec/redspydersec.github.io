¿Qué es Ghospyder?

Ghostpyder - X es una herramienta de fuzzing de línea de comandos diseñada para la detección rápida y eficiente de recursos ocultos (archivos, directorios, subdominios y Virtual Hosts) en auditorías de ciberseguridad. Está optimizada para entornos modernos, priorizando la rapidez y la detección inteligente sobre la fuerza bruta ciega.

Ventajas Competitivas

  • Velocidad Extrema (Go Lang): Al estar desarrollado en Go, aprovecha la concurrencia nativa para ejecutar miles de peticiones por segundo, superando a herramientas tradicionales basadas en Python o Bash en rendimiento puro.
  • Filtro Inteligente Anti-Ruido: A diferencia de otros fuzzers, realiza una comprobación previa y detecta el tamaño de la página de error (404/403) del servidor objetivo. Esto permite filtrar automáticamente las respuestas basura, presentando solo los resultados realmente interesantes.
  • Comprobación Previa Robusta: Evita "fuzzear al vacío" validando la conectividad y manejando errores de SSL/TLS (con la opción '-k') antes de iniciar el escaneo completo.
  • Escaneo Modular y Dirigido: Soporta modos específicos (DIR, DNS, VHOST) y exige la especificación explícita de extensiones** ('-e .php,.html'), garantizando que cada escaneo sea relevante y eficiente.

Modo de Uso Básico (DIR)

El uso es directo, con argumentos obligatorios para evitar escaneos innecesarios.

./ghostpyder-Linux-v2 -u <URL_Objetivo> -d <ruta_a_diccionario> -e <extensiones> [OPCIONES]

Ejemplo de Fuzzing de Directorios:

./ghostpyder-Linux-v2 -u https://objetivo.com -d /usr/share/wordlists/dirb/big.txt -e .php,.html,.js -k -r -fc 403 -fs 555

Explicación de Flags Avanzados:

  • -k: Omite la verificación de SSL (esencial en muchos entornos de prueba).
  • -r: Habilita el escaneo recursivo de directorios encontrados.
  • -fc 403: Excluye todas las respuestas con el código de estado HTTP 403 (Acceso Denegado).
  • -fs 555: Excluye respuestas con un tamaño específico (útil para eliminar la página de error base).

Lógica de Operación

La eficiencia de Ghostpyder se basa en un flujo de trabajo optimizado:

  1. Verificación Inicial: Una simple petición 'HEAD' a la raíz del objetivo. Si falla la conexión o SSL, el proceso se detiene antes de consumir recursos.
  2. Detección de Ruido: Una petición de prueba a una ruta aleatoria. El tamaño de la respuesta 404/403 se guarda automáticamente.
  3. Ejecución Concurrente: Múltiples hilos inician peticiones HTTP/DNS, respetando el límite de tasa ('-rate') para evitar bloqueos del servidor.
  4. Filtrado en Tiempo Real: Cada respuesta se compara con los códigos ('-fc') y los tamaños ('-fs' y el 404 detectado) antes de ser mostrada.
  5. Recursividad (Opcional): Si se encuentra un directorio 200/301, se añade a una cola de trabajo recursiva, iniciando un nuevo escaneo en esa ruta hasta el nivel de profundidad definido.
